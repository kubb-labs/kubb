Fix `banner`/`footer` bleeding into barrel files generated by `@kubb/middleware-barrel`.

When a plugin wrote a file (e.g. `index.ts`) with `output.banner` set (e.g. `'use server'`), `FileManager.mergeFile` was keeping the existing file's banner/footer instead of using the incoming file's values. This caused the barrel file generated at the same path to inherit the plugin's banner.

- `FileManager.mergeFile` now lets the incoming file (`b`) take precedence for `banner` and `footer` ("latest write wins").
- `middleware-barrel`'s `makeBarrel` explicitly sets `banner` and `footer` to `undefined` to ensure barrel files are always clean regardless of merge order.
